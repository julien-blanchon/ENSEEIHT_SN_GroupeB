Signe d’un entier

Énoncé

Le but de ce premier exercice est de comprendre comment exécuter un programme sous la plateforme PIXAL (Algorea). Lire complètement cette page. Les consignes sont plus bas.

En général, il y aura un énoncé qui décrit le programme à faire comme ceci :

Écrire un programme qui affiche « positif » si le nombre saisi par l’utilisateur est strictement positif, « negatif » si il est strictement négatif et « nul » si il est nul.

C’est le premier exercice sur PIXAL. Si vous êtes arrivé là, c’est que vous avez trouvé l’énoncé. Il comprend plusieurs rubriques :

  1. L’énoncé qui décrit le problème à résoudre (ce que vous êtes entrain de lire)
  2. Les entrées : les données sur lesquelles s’appuie votre programme (fournies par l’utilisateur ou le testeur)
  3. Les sorties : les données calculées par votre programme (et affichées à l’utilisateur ou au testeur)
  4. Des exemples : ils illustrent la description du problème et permettent de s’assurer de la bonne compréhension du problème à résoudre
  5. Un squelette de code : la structure du programme qu’il faudra respecter dans votre solution.
    • Il contient des parties à ne pas modifier.
    • Ce sont généralement celles qui gèrent l’interface avec l’utilisateur
    • saisie des informations (ici la valeur de la variable Nombre) et
    • l’affichage des résultats (ici l’affichage de la bonne chaîne de caractères).

À côté de l’onglet « ÉNONCÉ », un onglet « RÉSOLUTION » donne accès à un éditeur pour écrire le programme qui répond à l’énoncé.

Remarque : Si la fenêtre du navigateur est assez grande, l’énoncé et la résolution apparaissent côte à côte (et il n’y a pas d’onglet).

Les étapes dans la résolution d’un exercice sont les suivantes :

  1. Lire tout l’énoncé (énoncé, entrées, sorties, exemples, squelette)
  2. Normalement, l’éditeur de la partie « RÉSOLUTION » est initialisé avec le squelette du programme. Si ce n’est pas le cas, il faut le copier/coller.
  3. Compléter le programme
  4. Soumettre votre solution en cliquant sur « Valider la réponse ». Le vérificateur va (compiler puis) exécuter votre programme sur des tests (exemples) et vous dira s’ils réussissent ou non.
  5. Si des erreurs sont signalées par le vérificateur, il faut alors corriger le programme et soumettre à nouveau.
    • Il peut s’agit d’erreurs de syntaxe ou autre qui sont détectées par le compilateur (et donc avant l’exécution du programme). Il faut donc les corriger. Bien lire le message d’erreur qui est en général explicite et donne la ligne où l’erreur a été détectée, sa nature et parfois une proposition de correction.
    • Il peut s’agir d’erreurs lors de l’exécution du programme sur les tests prévus : votre programme ne fait pas ce qui est attendu. Le vérificateur compare ce qu’a affiché votre programme avec l’affichage attendu. Il vous signale en rouge le premier caractère qui ne coïncide pas.
  6. Si tous les tests réussissent, l’exercice est terminé et vous pouvez passer au suivant.

Pour cet exercice, nous vous demandons de :

  1. Lire les consignes
  2. Soumettre le code fourni dans le modifier pour voir les erreurs signalées
  3. Corriger le programme
    • Les premières sont des erreurs de compilation : bien lire les message d’erreurs indiqués et corriger les erreurs.
    • Ensuite, un des tests ne fonctionnera pas. Corriger le programme.
  4. Soumettre et recommencer en 3 si des erreurs restent

Remarque : Dans la partie « RÉSOLUTION » vous pouvez avoir plusieurs versions du programme (Code1, Code2, etc.). Il suffit de cliquer sur « + » pour ouvrir un nouvel éditeur et sur la croix « x » d’un éditeur pour le fermer.

À vous de jouer !

Entrée

Nombre : un entier

Sortie

une chaîne affichée, le signe de l’entier (“positif”, “negatif” ou “nul”)

Exemples

 Nombre  ->  resultat
-----------------
 5       ->  "positif"
-2       ->  "negatif"
 0       ->  "nul"

Squelette

with Ada.Text_IO;          use Ada.Text_IO;
with Ada.Integer_Text_IO;  use Ada.Integer_Text_IO;

----------------------------------------------------------------------
-- Attention, ce programme est volontairement faux.
-- Ne pas le corriger avant d'avoir vu les résultats affichés
-- par le testeur via « Valider la réponse ».
----------------------------------------------------------------------

-- Afficher le signe d'un entier lu au clavier
procedure Signe_Entier is
    Nombre: Integer -- Entier lu au clavier dont on veut connaître le signe

begin
    -- Demander un entier
    Get (Nombre)

    -- Afficher le signe de l'entier
    if Nombre > 0:
        Put ("positif");
    else
        Put ("negatif");
    end if;
    New_Line;

end Signe_Entier;
