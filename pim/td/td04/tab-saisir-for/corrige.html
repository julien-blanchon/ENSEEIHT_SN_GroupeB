Lire et afficher un tableau (la taille puis les éléments) : Corrigé possible

Voici quelques remarques sur la solution proposées ci-dessous.

  • (Lire) La variable Nb_Elements permet de savoir combien d’éléments doivent être lu et, en fin de sous-programme, s’il y a ou non dépassement de capacité et ainsi afficher le message après la saisie des éléments du tableau. On aurait pu utiliser Tab.Taille au lieu de Nb_Elements

  • (Afficher) Une autre solution aurait été de mettre le if dans le for pour savoir s’il faut afficher la virgule ou pas. L’inconvénient de cette solution est que l’on fait le test pour tous les éléments. Dans la solution proposée on ne fait qu’un seul test.

    On a particularisé le premier élément. On aurait pu particulariser le dernier.

Question 3 : Une autre solution pour que le programme principal (ou un autre sous-programme) utilisant Lire sache s’il y a eu dépassement de capacité est d’ajouter un nouveau paramètre en out (Depassement_Capacite) qui sera positionné ainsi :

Depassement_Capacite := Nb_Elements < Taille_Souhaitee

Le programme proposé.

with Ada.Text_IO;          use Ada.Text_IO;
with Ada.Integer_Text_IO;  use Ada.Integer_Text_IO;

-- Lire et écrire un tableau d'entiers.
procedure Tableau_IO is

    Capacite: constant Integer := 10;   -- Cette taille est arbitraire

    type T_TableauBrut is array (1..Capacite) of Integer;

    type T_Tableau is
        record
            Elements: T_TableauBrut;
            Taille: Integer;
            -- Invariant: 0 <= Taille and Taille <= Capacite;
        end record;


--------------------[ Ne pas changer le code qui précède ]---------------------


    -- Initialiser un tableau Tab à partir d'éléments lus au clavier.
    -- La nombre d'éléments est d'abord demandée, suivi des éléments.
    -- Les éléments en surnombre par rapport à la capacité du tableau
    -- sont ignorés et le message "Données tronquées" est affiché.
    procedure Lire (Tab: out T_Tableau) is
        Taille_Souhaitee: Integer;
        Nb_Elements: Integer;   -- Nombre d'éléments à lire
    begin
        -- Demander la taille
        Put ("Nombre d'éléments ? ");
        Get (Taille_Souhaitee);

        -- Déterminer le nombre d'éléments à saisir
        if Taille_Souhaitee > Capacite then
            Nb_Elements := Capacite;
        elsif Taille_Souhaitee < 0 then
            Nb_Elements := 0;
        else
            Nb_Elements := Taille_Souhaitee;
        end if;

        -- Demander les éléments du tableau
        for N in 1..Nb_Elements loop
            Put ("Element ");
            Put (N, 1);
            Put (" ? ");
            Get (Tab.Elements (N));
        end loop;
        Tab.Taille := Nb_Elements;

        if Nb_Elements < Taille_Souhaitee then
            Put_Line ("Données tronquées");
        else
            null;
        end if;
    end Lire;


    -- Afficher le tableau.  Les éléments sont affichés entre crochets, séparés
    -- par des virgules.
    procedure Ecrire(Tab: in T_Tableau) is
    begin
        Put ('[');
        if Tab.Taille > 0 then
            -- Écrire le premier élément
            Put (Tab.Elements (1), 1);

            -- Écrire les autres éléments précédés d'une virgule
            for I in 2..Tab.Taille loop
                Put (", ");
                Put (Tab.Elements (I), 1);
            end loop;
        else
            null;
        end if;
        Put (']');
    end Ecrire;


----[ Ne pas changer le code qui suit, sauf pour la question optionnelle  ]----

    Tab1: T_Tableau;    -- Un tableau
begin
    Lire (Tab1);

    -- Afficher le tableau lu
    Put ("Tableau lu : ");
    Ecrire (Tab1);
    New_Line;

end Tableau_IO;
