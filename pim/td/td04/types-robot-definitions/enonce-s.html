Robot de type 1 : définition des types

Énoncé

L’objectif de cet exercice est de modéliser un robot de type 1. Un tel robot se déplace dans un environnement qui peut être modélisé par un quadrillage dont chaque case correspond à une position possible du robot.

Un robot est donc caractérisé par son abscisse, valeur suivant l’axe des X, son ordonnée, valeur suivant l’axe des y, et sa direction (nord, sud, est ou ouest). La figure ci-après décrit un robot à la position x=4x = 4 et y=2y = 2, dans la direction « ouest ».

Les robots de type 1 ont des possibilités réduites qui se limitent à pivoter de 90° vers la droite et à avancer d’une case suivant sa direction courante.

Pour simplifier, on considère que le robot évolue dans un environnement infini sans obstacle.

Questions :

  1. Définir en language algorithmique les types nécessaires pour modéliser un robot de type 1.

  2. L’environnement dans lequel évolue le robot est en fait fini et comporte des obstacles. On suppose qu’un obstacle occupe une case du quadrillage. Il s’agit donc de savoir si la case est libre ou contient un obstacle.

    Définir en langage algorithmique un type pour modéliser l’environnement.

  3. Soit R1 un robot de type 1, comment faire pour :
    • obtenir son abscisse (et la stocker dans une variable X)
    • modifier sa direction pour lui donner la valeur nord
    • savoir si le robot R1 est bien sur une case libre de l’environnement E1

  4. Quels seraient les sous-programmes à définir sur les robots de type 1 ? On donnera seulement leur spécification en langage algorithmique.

  5. Définir les types en Ada et vérifier qu’il n’y a pas d’erreur sur PIXAL.

  6. Compléter le programme principal. Il s’agit de :
    • mettre les instructions manquantes après les commentaires,
    • supprimer la déclaration de la constante A_REMPLACER puis remplacer les expressions contenant A_REMPLACER par la bonne expression,
    • remplacer les ... pour compléter la phrase qui les contient.

    Puis le valider avec PIXAL.
